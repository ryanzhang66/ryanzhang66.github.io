Scaled Dot Product Attention的计算过程可以概括为以下几个步骤：

1. 输入：
    - 查询（Query）矩阵 Q：形状为 (seq_len_q, d_q)，表示要计算注意力的查询向量序列。
    - 键（Key）矩阵 K：形状为 (seq_len_k, d_k)，表示用于建立注意力关联的键向量序列。
    - 值（Value）矩阵 V：形状为 (seq_len_v, d_v)，表示与查询相关联的值向量序列。
    - 注意力掩码（可选）：用于指示哪些位置需要被忽略的二进制矩阵。
    
1. 计算注意力分数：
    - 计算注意力得分矩阵 A：通过将查询矩阵 Q 与键矩阵 K 的转置相乘并按比例缩放得到。具体计算公式如下：  
        A = softmax((Q * K^T) / sqrt(d_k))
    
    其中，softmax 表示对每个位置的注意力分数进行归一化，使得它们的总和为1。除以 sqrt(d_k) 是为了缩放分数，以避免在较大的维度空间中出现梯度消失或梯度爆炸的问题。
    
3. 应用注意力权重：
    - 将注意力得分矩阵 A 与值矩阵 V 相乘，得到注意力加权的值矩阵 O。具体计算公式如下：  
        O = A * V
    
    这一步将注意力权重应用于值矩阵，使得与查询相关的值得到突出表示。
    
4. 输出：
    - 输出矩阵 O：注意力加权的值矩阵，表示经过注意力机制处理后的结果。

**在这里我用一个例子来演示一下（为了方便理解）**
当计算注意力得分矩阵时，我们可以通过一个简单的例子来说明其计算过程。假设我们有以下查询矩阵 Q 和键矩阵 K：

```c
Q = [[1, 2, 3],
     [4, 5, 6]]

K = [[7, 8, 9],
     [10, 11, 12],
     [13, 14, 15]]
```

具体的计算步骤如下：

1. 点积操作：  
    将查询矩阵 Q 与键矩阵 K 的转置相乘，得到临时的注意力得分矩阵 S。计算过程如下：
    
    ````
    S = Q * K^T
    
      = [[1, 2, 3],
         [4, 5, 6]] * [[7, 10, 13],
                       [8, 11, 14],
                       [9, 12, 15]]
    
      = [[50, 68, 86],
         [122, 167, 212]]
    ```
    
    在这个例子中，注意力得分矩阵 S 是一个 2x3 的矩阵，表示每个查询与每个键的相关性。
    
    ````
    
2. 缩放操作：  
    在注意力得分矩阵 S 上进行缩放操作，除以 sqrt(d_k)。假设我们假设键的维度 d_k 为 3，那么计算过程如下：
    
    ````
    A = S / sqrt(d_k)
    
      = [[50, 68, 86],
         [122, 167, 212]] / sqrt(3)
    ```
    
    注意力得分矩阵 A 在这一步被缩放，这样每个得分都被除以 sqrt(3)。
    
    ````
    
3. 归一化操作：  
    对注意力得分矩阵 A 进行 softmax 操作，将得分归一化为注意力权重。计算过程如下：
    
    ````
    A = softmax(A)
    
      = softmax([[50, 68, 86],
                  [122, 167, 212]])
    ```
    
    在这个例子中，我们使用 softmax 函数对注意力得分矩阵 A 进行归一化处理，得到注意力权重矩阵。
    ````

4. 注意力加权值的计算过程
	当计算注意力加权值时，我们使用注意力权重矩阵（注意力得分归一化后的结果）和值矩阵 V。以下是一个示例来说明注意力加权值的计算过程。
	
	假设我们有以下注意力权重矩阵 A 和值矩阵 V：
	
	```
	A = [[0.2, 0.3, 0.5],
	     [0.6, 0.1, 0.3]]
	
	V = [[1, 2, 3],
	     [4, 5, 6],
	     [7, 8, 9]]
	```
	
	具体的计算步骤如下：
	
	1. 注意力加权值的计算：  
	    将注意力权重矩阵 A 与值矩阵 V 相乘，得到最终的注意力加权值矩阵 O。计算过程如下：
	    
	    ````
	    O = A * V
	    
	      = [[0.2, 0.3, 0.5],
	         [0.6, 0.1, 0.3]] * [[1, 2, 3],
	                             [4, 5, 6],
	                             [7, 8, 9]]
	    
	      = [[3.8, 4.6, 5.4],
	         [3.9, 4.6, 5.3]]
	    ```
	    
       在这个例子中，注意力加权值矩阵 O 是一个 2x3 的矩阵，表示经过注意力机制加权后的结果。每个位置的值是对应位置的注意力权重与值的乘积的累加结果。
	    ````

注意力加权值矩阵可以用于多种任务和应用中，包括但不限于以下几个方面：

1. 注意力加权：注意力加权值矩阵表示了查询与值之间的相关性和权重分布。可以将注意力加权值矩阵应用于序列中的不同位置，以获得更关注或更重要的信息。例如，在机器翻译任务中，可以使用注意力加权值矩阵来确定源语言句子中每个单词对目标语言句子生成的贡献。
    
2. 上下文编码：注意力加权值矩阵可以用于对序列进行上下文编码。通过将值矩阵乘以注意力加权值矩阵，可以生成经过注意力机制加权的上下文表示。这对于自然语言处理任务中的上下文理解和表示非常有用。
    
3. 序列生成：注意力加权值矩阵可以在序列生成任务中用作生成模型的一部分。通过将值矩阵与注意力加权值矩阵相乘，可以得到针对每个查询的加权值表示。这在机器翻译、摘要生成等任务中非常常见。
    
4. 可视化和解释性：注意力加权值矩阵可以用于可视化和解释模型的决策过程。通过可视化注意力权重，可以理解模型在输入序列中关注的位置和重要的信息。这对于模型的解释性和可解释性非常有帮助。
    

总之，注意力加权值矩阵提供了关注和权重分配的信息，可以在多个任务中用于上下文编码、序列生成、可视化和解释性等方面，从而增强模型的性能和理解能力。

#### Padding掩码
在处理数据的时候，我们为了规整数据，我们在神经网络中使用Padding和Masking
padding和masking示例：
![[Pasted image 20231024194808.png]]

**padding**：对不定长的文本序列进行填充，变为定长的序列，以便神经网络进行批量化学习
处理序列数据时，每个样本具有不同的长度是很常见的。 考虑以下示例（标记为单词的文本）：
```
[
  ["Hello", "world", "!"],
  ["How", "are", "you", "doing", "today"],
  ["The", "weather", "will", "be", "nice", "tomorrow"],
]

```
进行词汇查询后，数据可能会被矢量化为整数，例如：
```
[
  [71, 1331, 4231]
  [73, 8, 3215, 55, 927],
  [83, 91, 1, 645, 1253, 927],
]

```
数据是一个嵌套列表，其中各个样本的长度分别为3、5和6。 由于深度学习模型的输入数据必须是单个张量（在这种情况下，其形状为例如（batch_size，6，vocab_size）的形状），因此，比最长的项目短的样本需要填充一些占位符值（或者，一个占位符）。 可能还会在填充短样本之前截断长样本）。

**masking**：指示定长的序列中哪些是原本文的数据，哪些是padding后的数据，以便神经网络区分
现在所有样本都具有统一的长度，必须告知模型该数据的某些部分实际上是填充的，应该忽略。 该机制正在掩盖。mask 是形状为（batch_size，sequence_length）的2D布尔张量，其中每个False条目表示在处理过程中应忽略相应的时间步长。

首先，![\large XX^T](https://latex.csdn.net/eq?%5Clarge%20XX%5ET)，一个矩阵乘以它的转置，会得到什么结果，有什么意义呢？
我们知道，矩阵可以看作由一些向量组成，一个矩阵乘以它自己转置的运算，其实可以看成这些向量分别与其他向量计算内积。（此时脑海里想起矩阵乘法的口诀，第一行乘以第一列、第一行乘以第二列......嗯哼，矩阵转置以后第一行不就是第一列吗？这是在计算第一个行向量与自己的内积，第一行乘以第二列是计算第一个行向量与第二个行向量的内积第一行乘以第三列是计算第一个行向量与第三个行向量的内积.....）

**向量的内积，其几何意义是什么？**
**表征两个向量的夹角，表征一个向量在另一个向量上的投影**

下面我们进入一个超级详细的案例：

我们假设 ![\large X = [x_1^T; x_2^T; x_3^T]](https://latex.csdn.net/eq?%5Clarge%20X%20%3D%20%5Bx_1%5ET%3B%20x_2%5ET%3B%20x_3%5ET%5D)，其中![\large X](https://latex.csdn.net/eq?%5Clarge%20X)为一个二维矩阵，![\large x_i^T](https://latex.csdn.net/eq?%5Clarge%20x_i%5ET)为一个行向量（很多教材默认是列向量，但为了方便读者理解，我还是写成行向量）。对应下面的图，![\large x_1^T](https://latex.csdn.net/eq?%5Clarge%20x_1%5ET)对应“早”字embedding（语言数字化的这个过程叫做 Word Embedding，中文名称叫做 “词嵌入”， 而转化后获得到的向量矩阵就叫做词向量）之后的结果，以此类推。

下面的运算模拟了一个过程，即![\large XX^T](https://latex.csdn.net/eq?%5Clarge%20XX%5ET)。我们来看看其结果究竟有什么意义。
![[Pasted image 20231025143711.png]]
首先，行向量分别与自己和其他两个行向量做内积（“早”分别与“上”“好”计算内积），得到了一个新的向量。我们回想前文提到的向量的内积表征两个向量的夹角，表征一个向量在另一个向量上的投影。那么新的向量有什么意义呢？是行向量在自己和其他两个行向量上的投影。投影值大和小又有什么意义呢？

投影值大，意味两个向量相关度高。

我们考虑，如果两个向量夹角是90°，那么这两个向量线性无关，完全没有相关性。

更进一步，这两个向量是词向量，是词在高维空间的数值映射。词向量之间相关度高表示什么？是不是在一定程度上（不是完全）表示，在关注词A的时候，应给予词B更多的关注？

上图展示了一个行向量运算的结果，那么矩阵的意义是什么呢？

矩阵是一个方阵，我们以行向量的角度理解，里面保存了每个向量和自己与其他向量进行内积运算的结果。

##### Q K V矩阵
 在我们之前的例子中并没有出现`Q` `K` `V`的字眼，因为其并不是公式中最本质的内容。![[Pasted image 20231025144149.png]]
 其实，许多文章里所谓的Q K V矩阵、查询向量之类的字眼，其来源都是与矩阵的乘积，本质上都是的线性变换。那么为什么不直接使用而要对其进行线性变换呢？

当然是为了提升模型的拟合能力，矩阵都是可以训练的，起到一个缓冲的效果。

如果你真正读懂了前文的内容，读懂了这个矩阵的意义，那么你也就理解了所谓查询向量这一类的字眼的含义。

的意义
 假设  都服从均值为0，方差为1的标准高斯分布，那么中元素的均值为0，方差为d。当d变得很大时，中的元素的方差也会变得很大，如果中的元素方差很大，那么的分布会趋于陡峭（分布方差大，分布集中在绝对值大的区域）。总结一下就是的分布会和d有关。因此中每个元素除以后，方差又变为了1。这使得的分布的陡峭程度和d成功解耦，从而使得Transformer在训练过程中的梯度值保持稳定。
